В чём разница между несколькими потоками и несколькими асинхронными задачами?
Разница — в модели выполнения
Коротко:
- Потоки это несколько независимых исполнителей кода.
- Асинхронные задачи это кооперативное переключение внутри одного (или нескольких) потоков
---
Потоки (Threads) 'то реальные потоки ОС.
- ОС создаёт отдельный поток
- У него свой стек
- Он может реально выполняться на другом ядре CPU
- Переключение делает ОС
Плюсы:
- Настоящий параллелизм (использует несколько ядер)
- Хорошо для CPU-нагрузки
Минусы:
- Дорогие по памяти (стек ~2MB по умолчанию)
- Переключение потоков дорого
- Нужна синхронизация (Mutex, Arc и т.д.)
---
Асинхронные задачи (async/await)
Это задачи, которые выполняются внутри runtime (например, Tokio)
```rust
tokio::spawn(async {
    // async задача
});
```
- Нет нового потока
- Нет отдельного стека
- Это state machine.
- Переключение происходит в await
