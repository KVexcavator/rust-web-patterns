Загрузка WASM на локальный компьютер
Модуль WASM для загрузки WASM-файла и взаимодействовия с ним
- kelnel: предоставляет структуры как для рабочей области wasm-lib, так и для рабочей области wasm-client, чтобы они могли компилировать одни и те же структуры для сериализации и десериализации структур и байтов
- wasm-lib: создаёт исполняемый файл WASM с некоторой функциональностью, используя структуры, переданные в исполняемый файл WASM
- wasm-client: загружает исполняемый файл WASM и взаимодействовует с ним, передавая данные программе WASM и извлекая результаты
===
Макрос #[repr(c)] гарантирует, что структура согласована с представлением C для совместимости с ABI, порядка полей и выравнивания/заполнения, что делает структуру памяти предсказуемой с точки зрения C.
===
Крейт bincode сериализует типы данных Rust в необработанные байты. Это означает, что другие программы на Rust смогут принимать эти байты и преобразовывать их обратно в типы данных Rust, но не программы на других языках. 
===
Крейт wasmtime — это среда выполнения, которая будет запускать и загружать бинарный файл WASM. Крейт wasmtime-wasi — это компоновщик между средой выполнения wasmtime и бинарным файлом WASM. preview1 относится к версии API, на которую мы ориентируемся. Например, если мы хотим записать файл, вывести сообщение в терминал или получить доступ к порту, нашему коду WASM потребуется вызвать функцию во среде выполнения, как системный вызов в операционной системе
===
